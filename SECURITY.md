# Security Policy

## Supported Versions
We support security fixes for the following release targets:

| Version | Supported |
| --- | --- |
| v1.0.0-rc1 | Yes |
| < v1.0.0 | No |

## Reporting a Vulnerability
Please do not disclose security vulnerabilities publicly through GitHub issues. Instead, report security issues by emailing the security team or leads.

All security reports should include:
- A detailed description of the vulnerability.
- Steps to reproduce or a working Proof of Concept (PoC).
- Potential impact assessments.

## Response and Timeline
- **Acknowledgement**: Within 48 hours of receipt.
- **Resolution Plan**: Within 7 business days.
- **Coordinated Disclosure**: Public advisories will be published alongside corrective updates.
